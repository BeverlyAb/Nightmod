(app! k & options)

Calls a single method on Gdx.app.

(app! :error "MYTAG" "An error occurred, so I'm logging it!")
Options
:add-lifecycle-listener^LifecycleListener listener
Adds a new {@link LifecycleListener} to the application. This can be used by extensions to hook into the lifecycle more easily. The {@link ApplicationListener} methods are sufficient for application level development.
:debug^String tag, ^String message
Logs a debug message to the console or logcat
:debug^String tag, ^String message, ^Throwable exception
Logs a debug message to the console or logcat
:error^String tag, ^String message
Logs an error message to the console or logcat
:error^String tag, ^String message, ^Throwable exception
Logs an error message to the console or logcat
:exit
Schedule an exit from the application. On android, this will cause a call to pause() and dispose() some time in the future, it will not immediately finish your application. On iOS this should be avoided in production as it breaks Apples guidelines
:get-application-listener
:get-audio
:get-clipboard
:get-files
:get-graphics
:get-input
:get-java-heap
:get-log-level
Gets the log level.
:get-native-heap
:get-net
:get-preferences^String name
Returns the {@link Preferences} instance of this Application. It can be used to store application settings across runs.
:get-type
:get-version
:log^String tag, ^String message
Logs a message to the console or logcat
:log^String tag, ^String message, ^Throwable exception
Logs a message to the console or logcat
:post-runnable^Runnable runnable
Posts a {@link Runnable} on the main loop thread.
:remove-lifecycle-listener^LifecycleListener listener
Removes the {@link LifecycleListener}.
:set-log-level^int log-level
Sets the log level. {@link #LOG_NONE} will mute all log output. {@link #LOG_ERROR} will only let error messages through. {@link #LOG_INFO} will let all non-debug messages through, and {@link #LOG_DEBUG} will let all messages through.

(defmacro app!
  [k & options]
  `(u/call! ^Application (Gdx/app) ~k ~@options))
